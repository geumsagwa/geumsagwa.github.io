# Homepage 코드 Public 전환 사전점검 결과 (2026-03-12)

## 점검 범위

- 경로: `project/homepage/map`
- 방식: 시크릿 패턴 스캔 + 민감 키워드 점검 + 코드 재검토

## 총평

- 판정: **즉시 Public 전환 비권장 (High 리스크 존재)**
- 핵심 원인: 서비스 롤 키 하드코딩

## 주요 발견사항

### High-01. 서비스 롤 키 하드코딩

- 파일:
  - `map/upload-history1.js`
  - `map/upload-history2.js`
  - `map/rename-storage.js`
- 내용: `SERVICE_ROLE_KEY`에 JWT 형태의 실제 키 값이 코드에 직접 포함됨
- 영향:
  - 저장소 공개 시 즉시 인증정보 유출
  - 스토리지 업로드/삭제 및 DB 수정 API 오남용 위험
- 권장 조치:
  1. Supabase service_role 키 즉시 rotate
  2. 키를 코드에서 제거하고 환경변수(`process.env.SUPABASE_SERVICE_ROLE_KEY`)로 대체
  3. `.env`를 ignore하고 `.env.example`만 유지

### Medium-01. 운영 API 직접 호출 스크립트 노출

- 파일:
  - `map/rename-storage.js`
  - `map/upload-history1.js`
  - `map/upload-history2.js`
- 내용: 스토리지/REST API 직접 호출 로직이 공개되면 공격 표면이 넓어짐
- 권장 조치:
  - 운영용 스크립트는 private repo 분리 또는 최소한 키 주입형으로 변경
  - 실행 전 사용자 확인 프롬프트/대상 화이트리스트 추가

### Low-01. 저장소 기본 보호장치 확인 불가

- 현재 `project/homepage`는 Git 저장소가 아님
- 영향: `.gitignore`, 브랜치 보호, PR 체크 강제 상태를 점검할 수 없음
- 권장 조치:
  - 실제 공개 대상 Git 저장소 경로를 확정한 뒤 동일 점검 재실행
