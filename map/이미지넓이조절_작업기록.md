# EPUB 지도 이미지 넓이 조절 — 작업 기록 (보류)

## 목표
지도 이미지의 넓이를 텍스트 행 길이와 정확히 맞추기

## 현재 상태
- 인라인 스타일: `float:left; width:100%` + SVG `max-width:88%`
- float 레이아웃은 정상 작동 (페이지 흐름 끊지 않음)
- **넓이 조절은 보류 중**

## 텍스트 CSS 기준
- `<p>` margin: `0px 20px 15px 20px` (좌우 20px)
- `<h3>` margin: `50px 0 30px 20px`
- CSS 파일: 1권 `OEBPS/Styles/writing.css`, 2권 `OEBPS/Styles/Style0001.css`

## 시도한 방법들과 결과

### 1. 인라인 margin으로 맞추기
- `margin-left:20px; margin-right:20px` → SVG에 width 미지정이라 오히려 줄어듦

### 2. padding + box-sizing
- div에 `padding:0 20px; box-sizing:border-box;` + SVG `width:100%`
- 왼쪽 치우침 발생 (text-align:center 누락)
- 추가 후에도 원래와 비슷해 보임

### 3. calc(100% - 40px)
- SVG에 `width:calc(100% - 40px); margin:0 auto`
- 결과가 88%와 거의 동일 (330px 기준: calc = 290px ≈ 88%)

### 4. CSS 클래스 방식 (.map-wrap)
- writing.css에 `.map-wrap`, `.map-wrap svg` 클래스 추가
- `max-width: 88% → 92% → 94% → 97% → 102%` 변경 시도
- **변화 없음** — `max-width`는 SVG 실제 너비보다 크면 무효

### 5. max-width → width 변경
- `.map-wrap svg { width: 97% }` → 약간 커진 듯 보였으나
- `width: 102%`로 변경 → 변화 없음
- **CSS 클래스 방식 자체가 epub.js에서 제대로 반영 안 되는 것으로 추정**

## 핵심 발견사항
1. `max-width`는 SVG 실제 크기보다 큰 값이면 아무 효과 없음 → `width`를 써야 함
2. CSS 파일의 클래스 스타일이 epub.js 렌더링에서 제대로 적용되지 않는 현상 있음
3. 인라인 스타일은 작동함 (float 변경 등은 즉시 반영됨)
4. `margin`과 `max-width`는 독립적으로 누적됨 (겹치지 않음)

## 다음에 시도할 방향
1. **인라인 스타일로 `width` 직접 지정** — CSS 클래스가 아닌 인라인 `style="width:XX%"`
2. epub.js `rendition.themes.default()`에서 SVG 스타일 오버라이드 추가
3. reader.html에서 렌더링 후 JS로 SVG 크기 조절
4. epub.js DevTools(F12)로 실제 적용된 CSS 확인 필요
