# 코드쉴드 검토 리포트 (homepage, 2026-03-12)

## 범위

- 대상: `project/homepage`
- 방식: 정적 패턴 점검(시크릿/권한/스크립트 주입) + 코드 구조 리뷰

## 주요 이슈 (심각도 순)

### High-01. 관리자 권한 판정이 클라이언트 하드코딩에 의존 (조치 완료)

- 파일: `auth.js`, `admin.js`, `setup_members.sql`, `supabase-config.js`
- 조치 내용:
  - 개인 이메일 하드코딩/주입 방식 제거
  - `members(role/status)` 기반 관리자 판정(`isCurrentUserAdmin`)으로 전환
  - `admin.js`에서 비관리자 즉시 리다이렉트 + 인라인 `onclick` 제거
  - `setup_members.sql`에 RLS 정책(`self`, `admin`) 및 `is_admin_user()` 함수 추가
- 리스크:
  - 정책 SQL 미적용 환경에서는 기존 취약 상태가 유지됨
- 권장 조치:
  1. Supabase SQL Editor에서 `setup_members.sql` 실행
  2. `setup_members_seed_admin.sql` 실행으로 초기 관리자 1인 부여
  3. 운영 시 `setup_members_transfer_admin.sql` / `setup_members_revoke_admin.sql` 사용

### High-02. `map/` 운영 스크립트에 service role 키 하드코딩 (조치 완료)

- 조치 내용:
  - 하드코딩 키 제거
  - `map/_env.js` 공통 로더 추가
  - `map/.env.example` 제공 + `map/.gitignore`에 `.env` 추가
- 남은 조치:
  1. 기존 service role 키 즉시 rotate
  2. 운영 스크립트는 private 저장소/별도 배포 경로 분리

### High-03. 루트 마이그레이션 스크립트의 service role 키 하드코딩 (조치 완료)

- 파일: `migrate-essays.js`, `migrate-reviews.js`, `_env.js`, `.env.example`
- 조치 내용:
  - 마이그레이션 스크립트 하드코딩 키 제거
  - 루트 `_env.js`를 통해 `.env` 기반 로딩으로 전환
  - `.env.example` 제공으로 운영 표준화

### Medium-01. `innerHTML` 기반 렌더링 구간 다수 존재 (추가 완화 적용)

- 파일 예시:
  - `blog-detail-common.js` (sanitize 적용 완료)
  - `review.html` (sanitize 적용 완료)
  - `library.js` (book title/author 렌더링을 DOM API로 전환)
  - `diary.js` (본문 렌더링 escape 적용)
  - `board.js` (메모/답글 렌더링 escape 적용)
  - `admin.js` (status/id 화이트리스트 적용)
  - `auth.js` (템플릿 문자열 사용 구간 잔존)
- 리스크:
  - 외부 입력(게시글/리뷰 본문 등)이 섞이면 XSS 가능성 증가
- 권장 조치:
  - 사용자 입력/DB 본문 렌더링 시 sanitize 적용(적용 확대)
  - 단순 텍스트는 `textContent` 우선 사용

### Medium-02. CSP(Content Security Policy) 미적용 (조치 완료)

- 현황: `homepage` 내 전체 HTML(20개)에 CSP 메타 적용 완료
- 리스크:
  - `unsafe-inline` 유지 상태라 정책 강도는 중간 수준
- 권장 조치:
  - 인라인 스크립트를 외부 JS로 분리 후 `unsafe-inline` 제거(다음 단계)

## 참고(허용 가능 항목)

- `supabase-config.js`의 anon key 노출은 구조상 공개 전제
- 단, RLS 정책이 부실하면 데이터 노출로 이어질 수 있으므로 정책 점검 필수

## 우선순위 액션

1. `map/` service role 키 rotate + 환경변수화
2. `setup_members.sql` + 관리자 운영 SQL 세트 적용
3. 본문 렌더링 sanitize 적용
4. CSP 강도 상향(`unsafe-inline` 제거) 단계 진행
